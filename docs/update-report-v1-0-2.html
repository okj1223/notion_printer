변경사항

1. 현재 쓰지 않는 학습 데이터 관련 파일 제외

인쇄 결과 추천이나 학습용으로 쌓아둔 데이터 파일은 현재 팀 배포 단계에서 사용하지 않습니다. 1.0.2부터는 배포와 유지보수에 필요한 파일 위주로 정리했습니다.

2. 설치된 앱 버전 확인 기준 정리

설치 후 dpkg-query로 확인되는 패키지 버전과 앱 내부 버전 파일이 같은 값을 가리키도록 패키지 빌드 방식을 정리했습니다. 이후 버전에서도 같은 방식으로 확인하면 됩니다.

3. 배포 파일명 기준의 업데이트 안내

설치 파일명은 버전마다 달라지므로 안내 문구를 특정 버전에 고정하지 않고 1.x.x 예시로 바꿨습니다. 다운로드한 파일명에 적힌 버전을 넣어 실행하면 됩니다.

4. 업데이트 후 재시작 안내 위치 정리

업데이트 후 컴퓨터를 다시 시작해야 한다는 안내를 설치 안내 흐름 안으로 옮겼습니다. 설치 방법, 재시작, 버전 확인을 한 번에 따라갈 수 있게 정리했습니다.

5. ZIP 안의 HTML을 고를 때 페이지 구조를 먼저 확인합니다

ZIP을 풀고 HTML을 찾을 때 파일 확장자만 보지 않고, Notion 페이지 본문 구조인 .page-body가 있는지 확인합니다. 구조가 없으면 출력 후보에서 제외합니다.

6. 중첩 ZIP에서 정상 페이지 HTML만 남깁니다

ZIP 안에 다른 ZIP이 들어 있는 경우에도 자동 압축 해제는 유지합니다. 다만 압축을 푼 뒤 실제 페이지로 판단되는 HTML만 통합 출력에 넘깁니다.

7. Fast 품질 기본값을 빠름으로 맞췄습니다

앱을 처음 열었을 때와 터미널에서 별도 옵션 없이 실행했을 때 모두 1200px / WEBP 68 기준을 사용합니다. 더 선명한 이미지가 필요하면 고급 옵션에서 선명 또는 고품질을 선택하면 됩니다.

8. 하루 공개 버전은 하나로 정리합니다

2026-04-28에 나온 수정사항은 1.0.2에 합쳤습니다. 팀원이 보는 최신 다운로드와 업데이트 노트는 이 버전 하나만 가리킵니다.

설치와 확인

  1. 업데이트/다운로드 페이지에서 notion-printer_1.0.2_all.deb를 받습니다.
  2. 삭제 후 재설치 또는 터미널 업데이트 중 하나를 선택합니다.
  3. 설치가 끝나면 컴퓨터를 다시 시작합니다.
  4. 다시 시작한 뒤 설치된 버전을 확인합니다.
cd ~/Downloads

# 삭제 후 다시 설치할 때
sudo apt remove notion-printer

# 삭제하지 않고 업데이트할 때
sudo apt install ./notion-printer_1.0.2_all.deb
dpkg-query -W -f='${Version}\n' notion-printer
확인 기준 설치된 버전이 1.0.2로 표시되는지 먼저 확인하세요. ZIP 입력 안정화와 Fast 품질 기본값 정리도 1.0.2 설치 파일에 함께 들어 있습니다.

간단 확인 체크리스트

  • dpkg-query 결과가 1.0.2로 표시되는지 확인
  • 앱 실행 시 Fast 품질 기본값이 빠름 (1200px / WEBP 68)인지 확인
  • Ubuntu 앱 메뉴에서 Notion Printer가 정상 실행되는지 확인
  • HTML 또는 ZIP 파일 1개로 출력본이 생성되는지 확인
  • 여러 HTML/ZIP 파일 선택 시 통합 출력본이 생성되는지 확인
