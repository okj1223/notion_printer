Notion Printer 1.0.3 업데이트 노트

1.0.3은 중첩 ZIP이 들어 있는 Notion export를 더 안정적으로 처리하는 버전입니다. ZIP 안에 원본 페이지가 아닌 조각 HTML이 섞여 있어도, 정상 Notion 페이지 HTML만 골라 출력 대상으로 사용합니다.

설치 파일 notion-printer_1.0.3_all.deb
핵심 변경 ZIP 내부의 잘못된 HTML 후보 제외
권장 사용 Notion에서 받은 ZIP을 그대로 선택
최신 설치 파일 받기 업데이트/다운로드로 가기 1.0.2 노트 보기

한눈에 보기

부분 HTML 제외 .page-body가 없는 HTML은 ZIP 안에서 출력 후보로 쓰지 않습니다.
중첩 ZIP 유지 ZIP 안에 또 ZIP이 들어 있는 Notion export도 계속 자동으로 풀어서 확인합니다.
정상 페이지만 사용 원본 Notion 페이지 구조가 확인된 HTML만 통합 출력에 넘깁니다.
현재 권장 버전 ZIP으로 입력하는 문서가 있다면 1.0.3 사용을 권장합니다.

문제 상황

일부 Notion export ZIP에는 실제 페이지 HTML과 함께, 페이지 일부만 담긴 조각 HTML이 같이 들어갈 수 있습니다. 이런 조각 HTML은 Notion 페이지 본문을 감싸는 .page-body 구조가 없어서 통합 출력 단계에서 멈출 수 있었습니다.

기존 증상 ZIP을 선택했는데 Could not locate a Notion page container 또는 Could not locate .page-body 안내가 나오면, ZIP 안의 부분 HTML이 출력 대상으로 잘못 잡힌 경우일 수 있습니다.

변경사항

1. ZIP 안의 HTML을 고를 때 페이지 구조를 먼저 확인합니다

ZIP을 풀고 HTML을 찾을 때 파일 확장자만 보지 않고, Notion 페이지 본문 구조인 .page-body가 있는지 확인합니다. 구조가 없으면 출력 후보에서 제외합니다.

2. 중첩 ZIP에서 정상 페이지 HTML만 남깁니다

ZIP 안에 다른 ZIP이 들어 있는 경우에도 자동 압축 해제는 그대로 유지됩니다. 다만 압축을 푼 뒤 실제 페이지로 판단되는 HTML만 통합 출력에 넘깁니다.

3. 생성된 출력 HTML 재입력 방지는 그대로 유지합니다

기존처럼 *_print*.html 결과물이나 Notion Printer가 만든 관리 폴더 안의 HTML은 입력으로 다시 사용하지 않습니다. 원본 Notion export HTML 또는 ZIP을 선택하는 흐름을 유지합니다.

4. ZIP 입력 실패 가능성을 줄였습니다

정상 페이지 HTML이 ZIP 안에 있으면 부분 HTML 때문에 전체 작업이 멈추는 상황을 줄였습니다. 특히 공유 페이지가 많거나 export 구조가 깊은 문서에서 도움이 됩니다.

달라지는 사용 흐름

상황 1.0.3 기준 안내
Notion export ZIP을 그대로 선택 ZIP 안에서 정상 Notion 페이지 HTML을 찾아 출력 대상으로 사용합니다.
ZIP 안에 조각 HTML이 섞여 있음 .page-body가 없는 HTML은 자동으로 건너뜁니다.
HTML 파일을 직접 선택 원본 Notion export HTML을 선택합니다. 생성된 출력 HTML은 입력으로 사용하지 않습니다.
문제가 계속 발생 선택한 ZIP 파일명과 오류 메시지를 함께 공유하면 확인이 빠릅니다.

설치와 확인

  1. 업데이트/다운로드 페이지에서 notion-printer_1.0.3_all.deb를 받습니다.
  2. 삭제 후 재설치 또는 터미널 업데이트 중 하나를 선택합니다.
  3. 설치가 끝나면 컴퓨터를 다시 시작합니다.
  4. 다시 시작한 뒤 설치된 버전을 확인합니다.
cd ~/Downloads

# 삭제 후 다시 설치할 때
sudo apt remove notion-printer

# 삭제하지 않고 업데이트할 때
sudo apt install ./notion-printer_1.0.3_all.deb
dpkg-query -W -f='${Version}\n' notion-printer
권장 확인 ZIP 입력에서 오류가 났던 문서는 1.0.3 설치 후 같은 ZIP을 다시 선택해 확인하세요. 정상 페이지 HTML이 ZIP 안에 있으면 부분 HTML은 건너뛰고 출력이 진행됩니다.

간단 확인 체크리스트

  • dpkg-query 결과가 1.0.3으로 표시되는지 확인
  • 문제가 있었던 ZIP 파일을 그대로 선택해 출력이 진행되는지 확인
  • HTML/ZIP 여러 개를 선택했을 때 통합 출력본이 생성되는지 확인
  • 결과 파일이 _notion_printer_integrated_factory/ 아래 생성되는지 확인
