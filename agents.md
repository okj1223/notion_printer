# agents.md

## 작업 원칙
- 모든 디버깅은 증상 재현 → 원인 추적 → 최소 변경 → 영향 검토 순으로 진행한다.
- 문서가 서로 얽힌 경우에도 수정 범위를 최소화하고, 근거 없는 전역 수정은 피한다.
- 사용자 문서 요청(한국어)과 코드 증거를 함께 사용해 판단한다.

## 환경 독립성
- 작업 환경(회사/집/로컬 PC)은 고정되지 않으므로 절대 경로, 특정 IDE 설정, 특정 네트워크 조건에 의존하지 않는다.
- 문맥은 저장소 내 파일, 실행 로그, 사용자 제시 정보로만 판단한다.
- 저장소 구조가 동일하면 동일한 진단 규칙을 적용한다.

## 문서 우선순위 (충돌 시)
1. 현재 수정 중인 이슈와 직접적으로 관련된 최신 보고서/메모
2. `FIX_PASS_02_REPORT_KO.md`
3. `FIX_PASS_01_REPORT_KO.md`
4. 스키마 계약 파일
5. 핵심 실행 코드

## 프로젝트 핵심 파일
- 진입점/CLI: `notion_print_export.py`
- 실행/렌더/상태 처리: `notion_print_export/runtime.js`
- 학습/규칙 처리: `notion_print_export/learning.js`
- 스키마/데이터 계약: `learning_data/schemas/layout_event.schema.json`
- 보조/레포트: `FIX_PASS_*.md`

## 디버깅 규칙
- 재현 가능한 증거(로그, 예외, 입력값)를 우선 확보한다.
- 원인 미확인 상태에서 대규모 리팩터링은 금지한다.
- 수정 전/후 차이를 파일 단위로 좁혀 가며 진행한다.
- 레거시 동작 유지가 필요한 지점을 먼저 확인하고, 우회 처리 없이 정석 경로를 우선한다.
- 다중 수정이 필요한 경우,
  - 영향도가 낮은 수정부터 적용하고,
  - 각 변경의 의도를 기록한다.

## 출력물/엔진 디버깅 원칙
- 생성된 `*_print*.html` 파일은 디버깅용 재현 샘플이지, 최종 소스 오브 트루스가 아니다.
- 실제 수정은 항상 템플릿/엔진 파일에서 먼저 수행한다:
  - `notion_print_export/runtime.js`
  - `notion_print_export/theme.css`
  - `notion_print_export.py`
- 생성 HTML 직접 패치는 “즉시 확인용 임시 반영”으로만 사용하고, 확인 후 반드시 템플릿에도 같은 수정이 존재해야 한다.
- 특정 문서 1개만 기준으로 결론 내리지 않는다. 최소 3종의 대표 fixture를 기준으로 본다:
  - 이미지/미디어 중심 문서
  - 표/리스트/문단 중심 문서
  - 혼합형 문서
- UI 관련 이슈는 기능 삭제보다 `mode` 또는 `flag`로 분리하는 것을 우선한다.
- 출력물 비교 전 반드시 `manifest`의 아래 필드를 확인한다:
  - `generated_at`
  - `ui_default_mode`
  - `template_hashes`
- UI 디버깅은 실제 출력 경로를 기준으로 한다. 디버그 전용 정적 HTML을 따로 꾸미지 말고, 항상 `scripts/print_debug_factory.py`로 생성된 최신 결과를 본다.
- 디버깅 기본 진입 순서:
  - `python3 scripts/print_debug_factory.py --count 1`
  - 생성된 `Last preview` URL 확인
  - 그 URL에서 실제 화면 증상 재현
- 화면이 “잠깐 바뀌었다가 원본처럼 돌아가는” 류의 이슈는 코드만 보지 말고 실제 브라우저 상태를 확인한다:
  - Chrome headless 스크린샷을 로드 초반/지연 시점 두 번 캡처해 비교한다.
  - 필요하면 CDP로 computed style, 부모 체인, 주입된 stylesheet를 직접 확인한다.
  - print 전용 규칙이 screen으로 새는지 먼저 의심한다.
  - 편집기 UI가 `.pagedjs_pages` 안으로 빨려 들어갔는지 확인한다.
- 편집기 배너/패널/런처처럼 문서 바깥에 남아야 하는 UI는 `data-pagedjs-ignore="true"`를 유지한다.
- UI 디버깅 시 현재 기본 모드는 `full`이다. 비교가 필요할 때만 `?np_ui=minimal` 또는 다른 모드 쿼리를 명시해서 본다.
